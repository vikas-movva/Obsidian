# Question 1
We want to show that the three versions of the Vertex Cover problem are **polynomial-time Turing equivalent**:
- **VC-D (Decision)**
- **VC-O (Optimization)**
- **VC-F (Function / Find a solution)**

To do this, we show the following reductions:
- VC-D Ōēż VC-O
- VC-O Ōēż VC-D
- VC-F Ōēż VC-D
By transitivity, all three are equivalent.

---

## 1. Reducing VC-O to VC-D
Since `k` is between `0` and `|V|`, we can use **binary search**.

```python
function VC-O(G = (V, E)):
    lo ŌåÉ 0
    hi ŌåÉ |V|
    while lo < hi:
        mid ŌåÉ ŌīŖ(lo + hi) / 2Ōīŗ
        if VC-D(G, mid) = TRUE:
            hi ŌåÉ mid          
        else:
            lo ŌåÉ mid + 1      
    return lo
```

The predicate `VC_D(G, k)` is monotone so:
- If a vertex cover exists for size `k`, then it also exists for any `k' > k`.
Therefore binary search correctly finds the smallest valid `k`, which is the optimal solution.\

**Complexity**: 
Binary search makes `O(log |V|)` calls to VC-D and is therefore polynomial

---

## 2. Reducing VC-D to VC-O

```python
function VC-D(G = (V, E), k):
    n* ŌåÉ VC-O(G)
    return (n* Ōēż k)
```

If we already know the minimum vertex cover size `k*`, then:
- There exists a cover of size Ōēż `k` if and only if `k* Ōēż k`

**Complexity**: 
This function only has a single call to VC-O and is therefore polynomial

---

## 3. Reducing VC-F to VC-D
```python
function VC-F(G = (V, E)):
    k* ŌåÉ 0
    lo ŌåÉ 0,  hi ŌåÉ |V|
    while lo < hi:
        mid ŌåÉ ŌīŖ(lo + hi) / 2Ōīŗ
        if VC-D(G, mid) = TRUE:
            hi ŌåÉ mid
        else:
            lo ŌåÉ mid + 1
    k* ŌåÉ lo

    S        ŌåÉ Ōłģ               
    H        ŌåÉ G               
    budget   ŌåÉ k*              

    for each vertex v Ōłł V:
        if v Ōłē V(H):          
            continue
        H' ŌåÉ H with v and all edges incident to v removed
        if VC-D(H', budget ŌłÆ 1) = TRUE:
            S      ŌåÉ S Ōł¬ {v}   
            H      ŌåÉ H'
            budget ŌåÉ budget ŌłÆ 1
        else: do not add v
    return S
```

At every step, we are trying to build a valid minimum vertex cover of size `k*`.
For each vertex `v`:
- **If VC-D(H', budget - 1) is TRUE:**
    - Then we can include `v` in the solution
    - The remaining graph can still be covered within the remaining budget
    - So adding `v` is safe
- **If VC-D(H', budget - 1) is FALSE:**
    - Then no optimal solution includes `v`
    - So we skip it, and its neighbours must cover its edges later

## Problem 1: Clubs
To show Clubs is in NP, we need a polynomial-time verification algorithm.

Certificate: $S \subseteq {1,\dots, n}$

```python
Verify-Clubs(persons {1,ŌĆ”,n}, collection C, integer k, certificate S):
    if |S| > k:
        return FALSE
    for each club C_j Ōłł C:
        found ŌåÉ FALSE
        for each person p Ōłł C_j:
            if p Ōłł S:
                found ŌåÉ TRUE
                break
        if found = FALSE:
            return FALSE
    return TRUE
```

The algorithm checks the size constraint: `|S| Ōēż k` and if every club has at least one member in `S`

**Complexity**:
We have to check every club and every member inside it. Let `M = total number of memberships = ╬Ż |C_j|` 
`O(M┬Ęk) (or O(M) with a hash set)`
This is polynomial in the input size.

We can reduce from Vertex Cover, which is NP-complete.

**Reduction**:
We can convert a Vertex Cover instance into a Clubs instance.
```python
Reduce-VC-to-Clubs(G = (V,E), k):
    Persons ŌåÉ V                           
    C       ŌåÉ { {u, v} : (u,v) Ōłł E }
    return (Persons, C, k)
```
Each vertex ŌåÆ person
Each edge ŌåÆ club of size 2

**Correctness**:
If VC has a solution

Suppose there is a vertex cover `S` with `|S| Ōēż k`.
- Every edge `(u, v)` has at least one endpoint in `S`
- So each club `{u, v}` contains someone from `S`

Therefore, `S` is a valid Clubs solution.

If Clubs has a solution
Suppose there is a subset `S` with `|S| Ōēż k` that hits every club.
- Every club `{u, v}` contains someone from `S`
- So for every edge `(u, v)`, at least one of `u` or `v` is in `S`

This is the same as vertex cover.

**Complexity**: 
- One club per edge
- Total size: `O(|V| + |E|)`
- Reduction is linear ŌåÆ polynomial


Therefore Clubs Ōłł NP and Vertex Cover ŌēżŌéÜ Clubs so, Clubs is NP-complete

## Problem 2: EvenSplit
To show EvenSplit is in NP, we need a polynomial-time verification algorithm.

Certificate: $S \subseteq {1,\dots,n}$
```python
Verify-EvenSplit(integers a_1,ŌĆ”,a_n, certificate S):
    sum_S    ŌåÉ 0
    sum_comp ŌåÉ 0
    for i = 1 to n:
        if i Ōłł S:
            sum_S    ŌåÉ sum_S    + a_i
        else:
            sum_comp ŌåÉ sum_comp + a_i
    return (sum_S = sum_comp)
```

The algorithm checks whether the sum of elements in `S` is equal to the sum of elements not in `S`.

**Complexity**:  
We do one pass over all `n` elements.  
`O(n)` arithmetic operations (polynomial in input size).

---
We can reduce from Subset Sum, which is NP-complete.

**Reduction**:  
We convert a Subset Sum instance into an EvenSplit instance.
```python
Reduce-SubsetSum-to-EvenSplit(b_1,ŌĆ”,b_m, T):
    B = sum of all b_i
    
    a_1,ŌĆ”,a_m = b_1,ŌĆ”,b_m
    a_{m+1} = T + 1
    a_{m+2} = B - T + 1
    
    return (a_1,ŌĆ”,a_{m+2})
```

We keep the original numbers and add two extra values to force the total sum to split evenly.

**Correctness**:
First compute total sum:
- $A = B + (T+1) + (B-T+1) = 2B + 2$
- Target for each side is $A/2 = B + 1$

Also note:
- $(T+1) + (B-T+1) = B + 2 > B+1$  
    So both added numbers cannot be on the same side. They must be split.

If Subset Sum has a solution
Suppose there exists a subset `T` such that:
- $\sum_{i \in T} b_i = T$

Construct:
- `S = T Ōł¬ {m+2}`

Then:
- $\text{sum}(S) = T + (B - T + 1) = B + 1$

So this is a valid EvenSplit solution.

---
If EvenSplit has a solution suppose there exists `S` such that sum is `B + 1`.

Exactly one of the two added elements is in `S`.
**Case 1:** $a_{m+2} \in S$
- $\sum(T) + (B - T + 1) = B + 1$
- $\Rightarrow \sum(T) = T$
So we get a valid Subset Sum solution.

**Case 2:** $a_{m+1} \in S$
- $\sum(T') + (T + 1) = B + 1$
- $\Rightarrow \sum(T') = B - T$
So the complement of `T'` sums to `T`, giving a valid solution.

**Complexity**:
- We add 2 numbers and compute a sum
- Time: `O(m)`
- Output size is linear
So the reduction is polynomial.

Therefore EvenSplit Ōłł NP and Subset Sum ŌēżŌéÜ EvenSplit, so EvenSplit is NP-complete.

## The Reduction

The standard reduction from 3-SAT to ISD works as follows:

```python
Reduce-3SAT-to-ISD(phi with clauses C_1,...,C_k):
    V' ŌåÉ empty
    E' ŌåÉ empty
    for each clause C_j = (l_j1 Ōł© l_j2 Ōł© l_j3):
        add nodes (l_j1, j), (l_j2, j), (l_j3, j) to V'
    for each clause C_j:
        add edges (l_j1,j)-(l_j2,j), (l_j1,j)-(l_j3,j), (l_j2,j)-(l_j3,j) to E'
    for each pair of nodes (l_ja, j) and (l_ib, i) where i ŌēĀ j:
        if l_ja and l_ib are complementary (e.g. x and ┬¼x):
            add edge (l_ja, j)-(l_ib, i) to E'

    k ŌåÉ number of clauses
    return (G' = (V', E'), k)
```

**Key ideas:**

- Each clause becomes a **triangle** ŌĆö you can pick at most one node per clause
- Conflict edges ensure you cannot pick **both** $x_i$ and $\neg x_i$
- An independent set of size $k$ = one true literal picked per clause = satisfying assignment
